Most of the data from this was gotten from running the Girl 7 and 8 cards through [this tool](https://github.com/HunterRDev/eReader-Compression). However, as it did not properly separate the header and vpk from both of the second strips, I manually went through the game many times to get the structure and aditional data. What I found is that the first three lines of the fortune are taken in order one line each from 3 pools of 7 lines for each version of the game. Here, I have combined both games, so the first 7 lines in [the file line1](line1) are from Fortune Telling A, and the last 7 lines are from Fortune Telling B. The same applies to the files [line2](line2) and [line3](line3). The middle section of the fortune is the same and somewhat unfitting as a fortune, so I've omitted them, and will insert a blank line instead. The last four lines of the fortune, however, seem to be selected all from an ordered pool of 32 lines each for A and B, and displayed in the appropriate order. As I am combining both games, I have interleaved these pools in [the file line5678](line5678), such that odd line numbers are from Fortune Telling A, and even line numbers are from Fortune Telling B. Then I wrote a simple bash script to generate a fortune file from these 4 source files and pregenerated one as well. As I had to run the eReader programs many times to get additional data, if I am missing any lines, or if I am misunderstanding how they generate their fortunes, please let me know, preferably with a way for me to check for myself (such as which card, when scanned, generates the missing lines). You can run [randomizeKatrina.sh](randomizeKatrina.sh) to generate a new [katrinaEReader fortune file in the base directory](../katrinaEReader) with a default of 50 fortunes, though you can specify a different number of fortunes as the first argument, if you wish. You can also run `make randomize` in the base directory to generate a new 50-fortune [katrinaEReader file](../katrinaEReader).
